Technical Stack · Non-Technical Founder Edition

What you need to build
ESG Narrative Intelligence

Designed for a solo non-technical founder. Every tool chosen for maximum capability with minimum engineering. No code required for MVP — real code only when revenue justifies it.
01 — Client Portal (What clients see & use)
🖥️ Frontend & Client Interface
Where clients upload reports, review outputs, request changes
Start here
MVP Core 🫧
Bubble.io
Full no-code web app platform. Build the entire client portal without writing a line of code.
Drag-and-drop UI builder. Handles user login, file upload, dashboards, workflows. Used by funded startups at Series A.
Alternative 🔷
Framer / Webflow
Marketing site + simple client onboarding pages.
If you only need a landing page + intake form for early pilots, this is faster than Bubble.
Free tier 📋
Tally / Typeform
Client intake forms. Collects report uploads, company info, communication goals.
For the first 2 pilots you don't need a full portal — a smart form is enough to start.
Optional 🔔
Intercom / Crisp
In-app chat for client communication during review iterations.
Human-in-the-loop needs a communication channel. Chat inside the platform beats email chaos.
02 — AI Brain (The core intelligence)
🧠 AI & LLM Layer
Document analysis, claim separation, narrative generation, compliance checking
Differentiator
Core AI ⚡
Claude API (Anthropic)
Primary LLM for document analysis, claim categorisation, narrative generation.
Best-in-class for long document analysis and nuanced language tasks. Strong at following complex compliance instructions.
Orchestration 🔗
n8n / Make.com
Connects all tools together. Triggers AI pipeline when client uploads document.
No-code workflow automation. This is the "nervous system" that connects upload → AI → human review → output.
Knowledge Base 📚
Pinecone / Weaviate
Vector database storing EU regulations. AI checks claims against this knowledge base in real time.
This is what makes the regulatory check possible. All 5 EU frameworks are stored here and updated when regulation changes.
Alternative 🤖
OpenAI GPT-4o
Secondary or fallback LLM. Can be used for specific tasks where GPT-4o performs better.
Good for structured JSON outputs and function calling. Use alongside Claude, not instead of it.
03 — Document Processing
📄 Ingestion & Output Generation
Reading ESG reports in, generating compliant content out
No-code possible
Ingestion 🔍
Reducto / LlamaParse
Converts complex ESG PDFs into clean structured text that AI can process accurately.
ESG reports are messy — tables, charts, footnotes. These tools extract content far better than basic PDF readers.
Storage ☁️
Google Drive / S3
Stores uploaded ESG reports and generated output documents.
Start with Google Drive (free, integrates with everything). Move to AWS S3 when you scale.
Output Docs 📝
DocuGenerate / Carbone
Generates formatted Word/PDF compliance reports and narrative frameworks from AI outputs.
Clients expect polished documents, not raw AI text. This turns AI output into professional deliverables.
Video 🎬
HeyGen / Runway
Generates AI video prototypes from approved ESG narratives and scripts.
Your paid pilot was a 90-second ESG film. This keeps that capability in the platform at scale — no full production needed.
04 — Human-in-the-Loop Infrastructure
🤝 Expert Review Layer
The mandatory human gates — your key differentiator from pure AI tools
Differentiator
Review ✅
Notion / Airtable
Expert review dashboard. Shows AI outputs with approve / reject / modify actions for each claim.
For MVP, a structured Notion database is enough to manage human review before building a custom interface.
Collaboration 💬
Slack
Internal communication between you, ESG expert, and narrative specialist during review.
Free, fast, integrates with n8n so you get automatic notifications when a document needs human review.
e-Signature ✍️
DocuSign / Signeasy
Client signs off on the final compliance-checked narrative framework before content is published.
Creates legal paper trail. Protects client AND you. Essential for the compliance certificate deliverable.
Audit Trail 🗂️
Airtable
Logs every decision made at human review gates — who approved what, when, and why.
The compliance certificate needs to reference an audit trail. This is the database behind it.
05 — Business Infrastructure
⚙️ Payments, Auth & Analytics
The boring but essential layer
~1 day setup
Payments 💳
Stripe
Handles SaaS subscriptions, one-time payments for audits, invoicing.
Industry standard. Integrates with Bubble.io natively. You need this before your first paid pilot goes live.
Auth 🔐
Auth0 / Clerk
Secure login for clients. SSO, enterprise login for large corporate clients.
Enterprise ESG teams expect SSO. This is table stakes for B2B SaaS credibility.
Analytics 📊
PostHog / Mixpanel
Tracks how clients use the platform. Which steps take longest, where they drop off.
Essential for iterating the product. Free tier is more than enough for first 10 clients.
Email 📧
Resend / SendGrid
Automated emails: upload confirmed, review ready, report delivered.
Professional transactional email. Free tier covers early-stage volume easily.

⚠️ Founder's reality check
You don't need to build all of this to get your first two paying pilots. Phase 1 requires zero technical setup — just Claude, a shared Notion doc, and a Google Form. That's enough to charge €10k–15k and prove the model works.

The biggest risk for a non-technical solo founder is building too much too soon. Every tool you add before you have paying clients is overhead, not progress. The stack above is the full picture — but you only need the first three boxes to start.

The one thing worth investing in early: the Pinecone regulatory knowledge base. That's your defensible moat — it takes time to build and competitors can't copy it overnight.
